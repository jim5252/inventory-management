Architecture Overview

Factory Inventory Management System

A full-stack demo application: Vue 3 single-page frontend, Python FastAPI backend, and static JSON files standing in for a database. No persistence layer — all data is loaded into memory at server startup.

  • useAuth.js — mock current user, profile, tasks
  • useI18n.js — locale (en/ja), persisted to localStorage

Data Flow — Example: changing a filter

components/FilterBar.vue
User changes the Warehouse dropdown, bound via v-model to a shared ref from useFilters().
composables/useFilters.js
Because the ref is declared at module scope, every view importing useFilters() sees the update instantly — no prop drilling or event bus needed.
views/Dashboard.vue
A watch() on the four filter refs fires loadData(), which calls getCurrentFilters() and issues four parallel API calls via Promise.all.
src/api.js
Builds query params (skipping any value of 'all') and issues axios.get requests to http://localhost:8001/api/....
server/main.py
Route handler reads query params, filters the in-memory lists from mock_data.py, computes aggregates, and returns a Pydantic-validated response.
views/Dashboard.vue
Response updates raw data refs (summary, allOrders, inventoryItems). Vue's reactivity re-runs dependent computed() properties and re-renders KPI cards, tables, and SVG charts.

Known Gap

!
Frontend calls endpoints the backend doesn't implement

api.js defines getTasks, createTask, deleteTask, toggleTask, and createPurchaseOrder, targeting /api/tasks and /api/purchase-orders — but server/main.py has no routes for either. App.vue wraps these calls in try/catch, so they fail silently and the UI falls back to hardcoded mock data in useAuth.js. purchase_orders.json also exists but is empty.
